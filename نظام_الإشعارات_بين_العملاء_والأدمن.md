# 🔔 نظام الإشعارات بين العملاء والأدمن

## نظرة عامة
نظام إشعارات ثنائي الاتجاه يربط بين العملاء والأدمن، يتيح التواصل الفوري عند أي تحديث على الطلبات.

---

## ✨ الميزات الرئيسية

### 1. إشعارات الأدمن (Admin Notifications)
- **إشعار عند إلغاء الطلب**: عندما يلغي العميل طلبه، يصل إشعار فوري للأدمن
- **عرض تفاصيل الإشعار**: اسم العميل، رقم الطلب، قيمة الطلب
- **حالة القراءة**: تمييز الإشعارات المقروءة وغير المقروءة
- **عداد الإشعارات**: Badge يعرض عدد الإشعارات غير المقروءة
- **إدارة الإشعارات**: تعليم الكل كمقروء، مسح جميع الإشعارات

### 2. إشعارات العملاء (Customer Notifications)
- **إشعار عند تحديث الطلب**: عندما يحدث الأدمن حالة الطلب
- **إشعار عند الإلغاء**: تأكيد إلغاء الطلب
- **قسم خاص بالإشعارات**: تبويب Notifications في صفحة Profile
- **عداد الإشعارات**: Badge في القائمة الجانبية
- **تحديث تلقائي**: تحديث العداد كل 30 ثانية

## 🗂️ هيكل البيانات

### إشعار الأدمن (Admin Notification)
```javascript
{
    type: 'order_cancelled',           // نوع الإشعار
    orderId: 'ORD-1234567890',        // رقم الطلب
    customerName: 'أحمد محمد',         // اسم العميل
    customerEmail: 'customer@email.com', // بريد العميل
    orderTotal: 150.00,                // قيمة الطلب
    timestamp: '2026-03-13T10:30:00Z', // وقت الإشعار
    message: 'Customer أحمد محمد cancelled order #1234567890',
    isRead: false                      // حالة القراءة
}
```

### إشعار العميل (Customer Notification)
```javascript
{
    type: 'order_updated',             // نوع الإشعار
    orderId: 'ORD-1234567890',        // رقم الطلب
    newStatus: 'shipped',              // الحالة الجديدة
    timestamp: '2026-03-13T10:30:00Z', // وقت الإشعار
    message: 'Your order #1234567890 status has been updated to shipped',
    isRead: false                      // حالة القراءة
}
```

## 🔄 سيناريوهات الاستخدام

### السيناريو 1: إلغاء طلب من العميل
1. العميل يضغط "Cancel Order" في صفحة Profile
2. يتم تحديث حالة الطلب إلى `cancelled`
3. **إشعار للأدمن**: "Customer [Name] cancelled order #[ID]"
4. **إشعار للعميل**: "Your order #[ID] has been cancelled successfully"
5. تحديث عداد الإشعارات للأدمن

### السيناريو 2: تحديث حالة الطلب من الأدمن
1. الأدمن يغير حالة الطلب من Orders Tab
2. يتم تحديث حالة الطلب في النظام
3. **إشعار للعميل**: "Your order #[ID] status has been updated to [Status]"
4. تحديث عداد الإشعارات للعميل

## 📍 الملفات المعدلة

### 1. `profile.html`
- إضافة تبويب Notifications في القائمة الجانبية
- إضافة Badge لعدد الإشعارات غير المقروءة
- إضافة قسم Notifications Tab
- دوال الإشعارات:
  - `loadCustomerNotifications()` - تحميل إشعارات العميل
  - `markCustomerNotificationAsRead(index)` - تعليم إشعار كمقروء
  - `markAllCustomerNotificationsAsRead()` - تعليم الكل كمقروء
  - `clearAllCustomerNotifications()` - مسح جميع الإشعارات
  - `updateCustomerNotificationBadge()` - تحديث عداد الإشعارات
  - `getCustomerNotificationIcon(type)` - أيقونة الإشعار
  - `getCustomerNotificationColor(type)` - لون الإشعار
  - `formatNotificationTime(timestamp)` - تنسيق الوقت

### 2. `admin-script.js`
- تحديث دالة `updateOrderStatus` لاستخدام `updateOrderStatusWithNotification`
- دوال الإشعارات:
  - `loadAdminNotifications()` - تحميل إشعارات الأدمن
  - `markNotificationAsRead(index)` - تعليم إشعار كمقروء
  - `markAllNotificationsAsRead()` - تعليم الكل كمقروء
  - `clearAllNotifications()` - مسح جميع الإشعارات
  - `updateNotificationBadge()` - تحديث عداد الإشعارات
  - `updateOrderStatusWithNotification()` - تحديث حالة الطلب مع إشعار
  - `createCustomerNotificationFromAdmin()` - إنشاء إشعار للعميل

### 3. `admin-dashboard.html`
- إضافة زر Notifications في Sidebar مع Badge
- إضافة Notifications Tab
- أزرار "Mark All as Read" و "Clear All"

## 🧪 اختبار النظام

### اختبار 1: إلغاء طلب من العميل
1. سجل دخول كعميل
2. اذهب إلى Profile > My Orders
3. اضغط "Cancel Order" على طلب Pending أو Processing
4. تحقق من ظهور إشعار "Order cancelled successfully"
5. سجل دخول كأدمن (`aemade2016@gmail.com`)
6. تحقق من ظهور إشعار في Notifications Tab
7. تحقق من تحديث Badge في Sidebar

### اختبار 2: تحديث حالة الطلب من الأدمن
1. سجل دخول كأدمن
2. اذهب إلى Orders Tab
3. غير حالة طلب من Pending إلى Shipped
4. سجل خروج وسجل دخول كالعميل صاحب الطلب
5. اذهب إلى Profile > Notifications
6. تحقق من ظهور إشعار "Your order status has been updated to Shipped"
7. تحقق من تحديث Badge

### اختبار 3: إدارة الإشعارات
1. اضغط على إشعار غير مقروء (يتحول إلى مقروء)
2. اضغط "Mark All as Read" (جميع الإشعارات تصبح مقروءة)
3. اضغط "Clear All" (مسح جميع الإشعارات)

## 🔧 التخصيص

### إضافة نوع إشعار جديد
1. أضف النوع في دالة `getNotificationIcon()` أو `getCustomerNotificationIcon()`
2. أضف اللون في دالة `getNotificationColor()` أو `getCustomerNotificationColor()`
3. استخدم `createAdminNotification()` أو `createCustomerNotification()` لإنشاء الإشعار

### تغيير فترة التحديث التلقائي
```javascript
// في profile.html - السطر الأخير
setInterval(function () {
    updateCustomerNotificationBadge();
}, 30000); // 30 ثانية - غيرها حسب الحاجة
```
